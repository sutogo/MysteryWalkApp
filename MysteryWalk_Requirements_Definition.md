# お散歩アプリ「Mystery Walk」要件定義書

> **最終更新**: 2026-05-09  
> **バージョン**: v2.0（HANDOFF.md の確定事項を反映）

---

## 1. プロジェクト概要

目的地を伏せた状態で「方角」と「距離」のみを頼りに歩くことで、日常の風景に新しい発見（セレンディピティ）をもたらし、純粋な歩行体験を楽しむための **Android** アプリケーション。

ゲーミフィケーション（XP・レベル・バッジ）を最大の差別化要素とし、競合アプリ「Wherewalk」（iOS / 500円買い切り）にはない **継続利用の動機づけ** を提供する。

---

## 2. ターゲットユーザー

- 散歩が日課だが、コースがマンネリ化している人
- 意思決定コストを下げ、手軽に気分転換をしたい人
- ゲーム感覚で街歩きを楽しみたい人

---

## 3. 機能要件

### 3.1 探索・ナビゲーション機能

| 機能 | 詳細 |
|------|------|
| **現在地取得** | FusedLocationProviderClient によるリアルタイム緯度・経度取得 |
| **目的地ランダム抽出** | 指定半径（1km / 3km / 5km 等）から Overpass API でスポットを自動選定。カテゴリ（公園、カフェ、歴史的建造物等）フィルタリング対応 |
| **フォールバック** | スポットが見つからない場合はランダム座標を生成 |
| **ブラインド・ナビ** | 目的地名・地図表示の隠蔽。目的地までの直線距離表示 + 方角を示すコンパスUI |
| **到着検知** | 目的地から 50m 以内への進入を判定し通知 |

### 3.2 安全・補助機能

| 機能 | 詳細 | v1 対応 |
|------|------|---------|
| **振動フィードバックナビ（歩きスマホ防止）** | 方角に応じた振動パターンで画面を見ずにナビゲーション | ✅ |
| **帰還モード** | 出発地点または自宅への「帰り道用コンパス」切り替え | ✅ |
| **ネットワーク状態管理** | 圏外時ラストキャッシュで継続（目的地座標+GPS+コンパスのみ必要）。新規散歩開始時のみネットワーク必須 | ✅ |
| **お助けヒント（障害物回避）** | 直線上の大きな障害物（川、線路等）への間接的示唆 | ❌ v2 |

#### 振動パターン仕様

| 方向 | 角度範囲 | 振動パターン |
|------|----------|-------------|
| 正面 | ±15° | 短振動1回 |
| やや右/左 | 15°〜45° | 短振動2回 |
| 右/左 | 45°〜90° | 長振動1回 |
| 後方 | 90°超 | 連続短振動 |

- 振動間隔: **10秒ごと**（設定で変更可能）

### 3.3 ゲーミフィケーション・記録

| 機能 | 詳細 |
|------|------|
| **リワード表示** | 到着時に初めて目的地名・詳細情報を公開。アニメーション演出付き |
| **リワード写真** | Wikimedia Commons API から取得（取得不可ならカテゴリアイコンで代替） |
| **経験値(XP)システム** | 歩行距離や初訪問スポットの発見に応じてレベルアップ |
| **称号バッジ** | 探索カテゴリに応じたバッジ付与 |
| **履歴管理** | 過去に歩いたルートと発見した場所を地図上（osmdroid）で振り返り |

## 4. 非機能要件

### 4.1 UI/UX

- 屋外の直射日光下でも視認性が高いコントラスト
- ダークモード対応
- 片手で操作しやすいシンプルなインターフェース

### 4.2 バッテリー効率・GPS取得間隔

| 状態 | GPS取得間隔 |
|------|-------------|
| 通常時（バックグラウンド） | 30秒間隔（省電力） |
| 目的地 200m 以内 | 5秒間隔（高精度に自動切替） |
| フォアグラウンド時 | 5秒間隔 |

### 4.3 プライバシー

- 位置情報データの取り扱いに関する透明性の確保（保存場所・用途の明示）

### 4.4 オフライン対応

- v1 ではオフライン非対応
- 圏外時はラストキャッシュで継続（ナビに必要なのは目的地座標 + GPS + コンパスのみ）
- 新規散歩開始時のみネットワーク必須

## 5. 技術スタック

| カテゴリ | 技術 |
|----------|------|
| **言語** | Kotlin |
| **UI** | Jetpack Compose + Material 3 |
| **アーキテクチャ** | MVVM + Clean Architecture |
| **DI** | Hilt |
| **DB** | Room |
| **地図** | osmdroid（OSMベース、無料） |
| **位置情報** | FusedLocationProviderClient |
| **スポット検索** | OpenStreetMap (Overpass API) |
| **写真取得** | Wikimedia Commons API |
| **ネットワーク** | Retrofit + Moshi |
| **アニメーション** | Lottie Compose |
| **設定保存** | DataStore Preferences |
| **テスト** | JUnit5 + MockK + Turbine |

### 5.1 レイヤー構成

```
app/src/main/java/com/mysterywalk/app/
├── ui/           # Compose画面 (NavScreen, RewardScreen, HistoryScreen, ProfileScreen)
├── viewmodel/    # ViewModel (NavigationVM, GamificationVM, HistoryVM)
├── domain/       # UseCase (NavigateUC, FindSpotUC, CalculateXpUC)
├── data/         # Repository (LocationRepo, SpotRepo, UserRepo)
│   ├── local/    # Room DB (Entity, DAO)
│   └── remote/   # Retrofit (Overpass API, Wikimedia API)
├── service/      # フォアグラウンドサービス (LocationService)
├── di/           # Hilt Module
└── util/         # 計算ユーティリティ (距離, 方角)
```
